==============================================================
  YUNGE 个人 AI 助手 - 便携版
==============================================================

【启动】
  双击 _run_yunge.bat
  等待终端出现 YUNGE 的 Logo 字样
  浏览器打开:  http://localhost:6185

【登录】
  用户名: miku
  密码:   miku
  (初次进入若提示注册账号, 任意填一个即可)

【首次使用必须做的一步: 添加模型 API Key】
  登录 -> 设置 -> 模型提供商 / Provider
  选 OpenAI / DeepSeek / Anthropic 任一
  填 API Key, 保存
  回到对话页, 选刚刚添加的模型, 即可开始聊天

【演示流程参考】
  1. 对话: 直接发一句话, 看流式输出
  2. 知识库: 设置 -> 知识库 -> 上传一份 PDF/MD
     回到对话, 启用该知识库, 提问相关内容, 它会自动召回片段
  3. 画像 (Profile): 顶部菜单 "画像 / Profile"
     聊几轮后点 "生成 MBTI" 或查看时间线
  4. Agent: 对话中触发知识库检索 / 网络搜索时会显示工具调用

【关闭】
  关闭终端窗口即可

【目录结构】
  _run_yunge.bat   双击启动入口
  使用说明.txt     本文件
  main.py          主程序
  miku/            后端代码
  python/          内置 Python 3.12 (不要删)
  data/            你的对话/知识库数据 (可保留, 也可删了从零开始)

【常见问题】
  Q1: 浏览器显示 "无法访问"
  A:  等终端打印 YUNGE Logo 之后再打开. 启动需要 5-10 秒.

  Q2: 终端报错 "端口 6185 被占用"
  A:  先关掉占用 6185 的程序, 或者重启电脑再试.

  Q3: 对话没有任何回复
  A:  99% 是 API key 没填或者 key 余额不足.
      去 设置 -> Provider, 检查模型在线状态.

  Q4: 后续还想用, 数据会丢吗
  A:  不会. 关闭重开数据都在 data/ 下, 直接双击 _run_yunge.bat.
